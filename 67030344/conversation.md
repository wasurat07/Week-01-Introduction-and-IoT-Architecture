## คำตอบ
```
Internet of Things (IoT) คือการที่อุปกรณ์ต่างๆ เชื่อมต่อกับอินเทอร์เน็ตเพื่อรับส่งข้อมูลและสั่งการได้อัตโนมัติ
โดยประกอบด้วย เซนเซอร์ (เก็บข้อมูล), การเชื่อมต่อ (รับส่งข้อมูล), และ ระบบประมวลผล (วิเคราะห์และตัดสินใจ)
เพื่อเพิ่มความสะดวกสบายและประสิทธิภาพในการทำงาน

ตัวอย่างระบบ IoT ที่พบในชีวิตประจำวัน Smartwatch
เพราะ Smartwatch มีเซนเซอร์ตรวจวัดอัตราการเต้นของหัวใจและกิจกรรมต่างๆ
จากนั้นจึงเชื่อมต่อกับอินเทอร์เน็ตผ่านสมาร์ทโฟนเพื่อส่งข้อมูลไปยังแอปพลิเคชันหรือคลาวด์
เพื่อประมวลผลสุขภาพและแสดงผลให้ผู้ใช้ทราบได้แบบเรียลไทม์ครับ 
```

## คำตอบ
```
เทคโนโลยีในชั้น Internet Layer สำหรับ IoT ถูกออกแบบมาเพื่อแก้ปัญหาการเชื่อมต่ออุปกรณ์จำนวนมหาศาลที่มีข้อจำกัดด้านทรัพยากร
- IPv6 มีพื้นที่แอดเดรสขนาดมหาศาล รองรับจำนวนอุปกรณ์ IoT ที่เพิ่มขึ้นอย่างรวดเร็วทั่วโลกได้โดยไม่มีวันหมด
- 6LoWPAN เป็นเทคนิคที่ช่วยให้การส่งข้อมูล IPv6 ผ่านเครือข่ายไร้สายพลังงานต่ำ (เช่น IEEE 802.15.4)
ทำได้โดยการบีบอัดข้อมูลส่วนหัว (Header) ให้เล็กลง เพื่อประหยัดแบนด์วิธ
- RPL (Routing Protocol for Low-Power and Lossy Networks) เป็นโปรโตคอลการหาเส้นทางที่ออกแบบมาเพื่อเครือข่ายที่มีการเชื่อมต่อไม่เสถียร
โดยเน้นการประหยัดพลังงานในการส่งข้อมูลข้ามโหนดไปจนถึงจุดหมาย

ทำไมถึงเหมาะกับอุปกรณ์ IoT
- ประหยัดพลังงาน เทคโนโลยีเหล่านี้เน้นการส่งข้อมูลที่กระชับ ลดภาระการประมวลผลและการใช้พลังงานของแบตเตอรี่ในอุปกรณ์เซนเซอร์
- จัดการแบนด์วิธได้ดี การบีบอัด Header ช่วยให้ใช้ช่องสัญญาณน้อยลง ลดความแออัดของเครือข่าย ทำให้ส่งข้อมูลได้แม้ในสภาวะที่สัญญาณต่ำ
- รองรับขนาดเครือข่าย โครงสร้างที่ยืดหยุ่นช่วยให้ระบบรองรับการเชื่อมต่ออุปกรณ์หลักร้อยถึงหลักพันในพื้นที่เดียวได้อย่างเป็นระเบียบและเสถียร 
```

## คำตอบ
```
ความท้าทายหลักของระบบ IoT
ระยะทาง (Communication Range)
ต้องเลือกระหว่างเครือข่ายระยะใกล้ (เช่น Bluetooth/Zigbee) หรือระยะไกล (เช่น LoRaWAN/NB-IoT) ให้เหมาะกับพื้นที่ใช้งาน
แบนด์วิธ (Bandwidth)
อุปกรณ์ส่วนใหญ่มีแบนด์วิธจำกัด ต้องจัดการข้อมูลให้กระชับ เพื่อไม่ให้ช่องสัญญาณเต็มหรือล่าช้า
การใช้พลังงาน (Power Usage)
อุปกรณ์ IoT มักใช้แบตเตอรี่ การส่งข้อมูลที่ถี่เกินไปหรือการเชื่อมต่อตลอดเวลาจะทำให้แบตเตอรี่หมดเร็ว
การเชื่อมต่อเครือข่าย (Connectivity)
สภาพแวดล้อมจริงอาจมีจุดอับสัญญาณ อุปกรณ์ต้องมีความสามารถในการเชื่อมต่อใหม่เองได้ (Self-healing)
การทำงานร่วมกัน (Interoperability)
อุปกรณ์จากต่างผู้ผลิตมักมีมาตรฐานที่ต่างกัน ทำให้การนำมาใช้งานร่วมกันในแพลตฟอร์มเดียวทำได้ยาก
ความปลอดภัย (Security)
อุปกรณ์ IoT มักมีพลังประมวลผลต่ำ ทำให้การเข้ารหัสข้อมูลทำได้ยาก จึงตกเป็นเป้าโจมตีได้ง่าย

สถานการณ์ ระบบเซนเซอร์ตรวจจับที่จอดรถในเมือง
ความท้าทายที่ส่งผลต่อการออกแบบ
การใช้พลังงาน (Power Usage)
เซนเซอร์ที่ติดตั้งบนพื้นถนนไม่สามารถลากสายไฟได้ จึงต้องใช้แบตเตอรี่ที่ใช้งานได้นาน 3-5 ปี
วิศวกรจึงต้องเลือกใช้ โหมด Deep Sleep (ตื่นมาส่งข้อมูลวันละไม่กี่ครั้งหรือส่งเฉพาะตอนสถานะเปลี่ยน) เพื่อยืดอายุการใช้งาน
ระยะทางและการเชื่อมต่อ (Communication Range & Connectivity)
การติดตั้งเซนเซอร์นับพันจุดทั่วเมืองทำให้การใช้ Wi-Fi เป็นไปไม่ได้ เพราะระยะส่งสั้นและซับซ้อนเกินไป
นักออกแบบจึงต้องเปลี่ยนไปใช้เครือข่ายอย่าง LoRaWAN หรือ NB-IoT ที่ส่งสัญญาณทะลุสิ่งกีดขวางได้ดีและไกลกว่า
การทำงานร่วมกัน (Interoperability)
เซนเซอร์ซื้อจากต่างยี่ห้อกัน แต่ต้องการส่งข้อมูลเข้าสู่ Dashboard กลางของเมือง
ผู้ออกแบบจำเป็นต้องใช้มาตรฐานการสื่อสารที่เป็นสากล เพื่อให้ซอฟต์แวร์สามารถ "อ่านภาษาเดียวกัน" ได้
```
